# AugurForge Cerebras + Gemma 4 Win Plan

## Thesis

AugurForge must not read as a generic finance dashboard with AI sprinkled on top. It must read as:

> Gemma 4 understands messy actuarial inputs, Cerebras makes the agent loop feel instant, and the browser turns the result into a live quantitative sandbox.

The judges should be able to pause the demo at any second and see the stack:

- Gemma 4 is the intelligence: model selection, multimodal reading, generated model spec, risk narrative.
- Cerebras is the speed: visible TTFT, tokens/sec, per-agent completion times, fast cascade.
- AugurForge is the proof: deterministic math, 2D/3D/Animate render, risk flags, board-ready explanation.

## Track Positioning

Primary track: **Multiverse Agents**.

Reason: the app is explicitly a multi-agent, multimodal Gemma 4 system:

- Orchestrator routes the model.
- Modeler reads text plus image input.
- Visualizer creates the dashboard spec.
- Sensitivity explains movement after slider changes.
- Risk emits governance flags.
- Explainer streams entry or expert narrative.

Secondary story: **Enterprise Impact**.

Reason: actuarial and quant users understand the value of fast exploration, governance flags, and explainable scenario analysis. The product must stay framed as decision-support, not advice.

## SaaS Research Translation

The redesign borrows product principles from current high-end SaaS surfaces:

- **Notion AI** (`https://www.notion.com/product/ai`): positions AI as a teammate inside the work surface, with agents, enterprise search, usage controls, and trust language. AugurForge should make Gemma agents feel embedded in the workflow, not bolted on.
- **Linear** (`https://linear.app/`): presents a dense product-development system with side navigation, issue detail, activity, labels, and AI agents in the flow. AugurForge should feel fast, structured, and operational, with a precise left rail and a clear execution stream.
- **Figma AI** (`https://www.figma.com/ai/`): frames AI as a creative/productive workflow from explore to polish to ship, with agentic generation and code-backed outcomes. AugurForge should make generated models feel like a real creative act that becomes a working artifact.

Visual conclusion: use an elegant product workbench, not a sci-fi cockpit. Artistic means controlled composition, typography, spacing, and one or two memorable surfaces. It does not mean gradients everywhere.

## Demo Order

### 0-5s: Stack Promise

Show a composed product surface with the line:

> Gemma 4 turns actuarial input into a live model. Cerebras makes it feel instant.

The UI should already show:

- `Gemma 4 on Cerebras`
- `TTFT`
- `tokens/s`
- `6-agent cascade`
- `decision-support, not advice`

### 5-18s: Multimodal Modeler

Attach a loss-triangle or messy chart image.

Required visible cue:

- `Gemma 4 Modeler`
- `Vision input attached`
- `Inferred model + parameters`

Do not let this look like ordinary file upload. It is the track proof.

### 18-32s: Generated Model Moment

Use the prompt:

> Build a Black-Scholes option pricing sandbox with Greeks and a pricing curve.

Required visible cue:

- `Generated by Gemma 4`
- `Compiled to deterministic browser math`
- `No arbitrary generated JavaScript`

The new model must appear as an artifact, not a chat answer.

### 32-45s: Live Tweak Loop

Drag volatility or spot.

Required visible cue:

- Chart moves instantly.
- Metrics update client-side.
- Sensitivity, risk, and explainer stream after release.
- The UI shows that only interpretation calls went back to Gemma.

### 45-55s: Cerebras Speed Proof

Run speed race.

Required visible cue:

- Per-call TTFT.
- tokens/sec.
- total ms.
- baseline clearly labeled as real or simulated.

Do not fake the baseline. A simulated baseline is acceptable only if clearly labeled.

### 55-60s: Submission Line

End on:

> Six Gemma agents. Multimodal input. Generated model. Live simulation. Cerebras-speed loop.

## UI Redesign Requirements

The app should look like a serious SaaS workbench:

- Left rail: product identity, live/mock status, demo modes, and stack proof.
- Main stage: uploader/execution prompt, controls, chart, generated model badge.
- Right rail: speed HUD, metrics, agents, risk, explainer.
- Panels should be quiet and precise, not decorative cards stacked at random.
- Accent color must mean state or action, never decoration.
- Motion must be stateful: cascade starts, tokens stream, race fills, chart reveals.
- The first viewport must make Cerebras + Gemma visible without reading docs.

## Copy Rules

- Say `Gemma 4` wherever intelligence is shown.
- Say `Cerebras` wherever speed is shown.
- Say `deterministic browser math` wherever generated models are shown.
- Keep `decision-support, not advice` visible but quiet.
- Avoid generic claims like "AI-powered insights" unless the UI proves what happened.

## Failure Modes That Lose

- Monte Carlo is the only convincing model.
- The image input does not visibly matter.
- Generated model path looks like a prebuilt dropdown.
- Cerebras timing is hidden in a small badge.
- The UI looks like a default dark-mode admin template.
- Copy implies production reserving or financial advice.
- The speed race looks fake or unlabeled.

## Acceptance Bar

Before recording, the local demo must show:

- A clear generated Black-Scholes sandbox in mock mode.
- Visible per-agent cascade with Gemma 4 labels.
- Visible Cerebras speed panel with TTFT and tokens/sec.
- A visual distinction between generated runtime and library templates.
- A composed UI that would not embarrass us next to Linear, Notion, or Figma product screens.
- `npm run typecheck` and `npm run build` passing.
