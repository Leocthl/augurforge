# PRODUCT.md - AugurForge

## Register

product

## Product Purpose

AugurForge is an instant, interactive model sandbox for actuaries, quants, and hackathon judges evaluating Gemma 4 on Cerebras. A user can provide a model idea, dataset, chart, screenshot, or sketch, then watch a Gemma 4 agent cascade turn it into a live browser simulation with tunable parameters, visible risk flags, and a streamed explanation.

The product is decision-support, not financial advice. It proves a fast exploration loop, not a governed reserving or pricing system.

## Primary Users

- Actuaries and quantitative analysts exploring assumptions, tail risk, or pricing sensitivity.
- Insurance, finance, and enterprise judges looking for credible workflow impact.
- Hackathon judges looking for visible Gemma 4 intelligence, multimodal input, and Cerebras speed.

## Job To Be Done

Turn a vague or messy quantitative input into a live model fast enough that the user can ask, tune, and understand in one continuous loop.

## Demo Promise

Gemma 4 understands the input and shapes the model. Cerebras makes the agent loop feel immediate. The browser proves the output with deterministic math, a rendered simulation, and explanation panels that stream as each agent resolves.

## Core Capabilities To Keep Visible

- `Gemma 4` labels must appear where intelligence is happening: routing, vision, model generation, risk, sensitivity, explanation.
- `Cerebras` labels must appear where speed is happening: TTFT, tokens per second, speed race, fast cascade.
- Generated models must be presented as working artifacts, not chat text.
- The app must distinguish generated runtime templates from library templates.
- Image attachment must feel like multimodal modeling input, not a generic file upload.
- The phrase `decision-support, not advice` stays visible but quiet.

## Tone

Precise, calm, mathematically literate, and product-grade. The UI should feel like a serious modeling workbench with an elegant academic edge, closer to a SideKlick or Linear-quality product surface than a sci-fi control room.

Copy should be short and concrete. Avoid generic AI claims unless the interface proves the claim in the same viewport.

## Anti-References

- Neon finance dashboard.
- Purple-blue AI gradient shell.
- Dark cyber cockpit.
- Marketing landing page.
- Identical card grid.
- Chatbot-first app where the model appears as prose only.
- Copy that implies regulated financial advice or production reserving.

## Design North Star

The first viewport should read as a restrained light SaaS workbench: Geist typography, soft gray canvas, quiet frosted panels, a dark simulation viewport for contrast, and one disciplined blue accent for primary action and state.

The product should look calm enough for actuaries and sharp enough for a hackathon demo.
